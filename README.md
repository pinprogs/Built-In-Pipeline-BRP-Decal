# BRP 박스 데칼

이 프로젝트는 **Unity Built-in Render Pipeline** 환경에서 동작하는 간단한 **박스 기반 데칼 구현 예제**입니다.  
현재 데칼은 텍스처를 사용하지 않고, **단색(Color)** 만으로 화면에 투영됩니다.

## 특징

- 이 데칼은 **컬러 기반 박스 데칼**입니다.
- 데칼은 **박스 볼륨 내부에 포함되는 픽셀**에만 적용됩니다.
- 한 번 화면을 그릴 때 최대 **100개의 데칼**을 한 번에 처리할 수 있습니다.
- 이 최대 개수는 **코드에서 조정 가능**합니다.
- 현재 구현은 **텍스처를 지원하지 않습니다**.
- 데칼 색상은 **Color 값**으로만 제어됩니다.

## 동작 방식

이 프로젝트는 카메라의 **깊이 텍스처**를 이용해 화면상의 픽셀 위치를 다시 **월드 좌표**로 복원한 뒤,  
각 픽셀이 데칼 박스 내부에 들어오는지 검사해서 색을 덮어쓰는 방식으로 동작합니다.

즉, 데칼 메쉬를 직접 붙이는 방식이 아니라,  
**화면 공간(Screen Space)** 에서 박스 형태의 투영 영역을 계산해 데칼을 적용하는 구조입니다.

## 구현 개요

### `BRPDecal`
각 데칼 오브젝트의 색상과 `Transform` 정보를 관리합니다.

### `BRPDecalManager`
등록된 데칼들을 모아 `CommandBuffer`를 구성하고 렌더링합니다.

### `BRPDecal.shader`
깊이 텍스처를 바탕으로 월드 좌표를 복원한 뒤 박스 내부 여부를 판정하여 데칼 색을 적용합니다.

## 제한 사항

- **텍스처 데칼은 지원하지 않습니다.**
- 현재는 **단색 데칼만 지원합니다.**
- 데칼 수가 많아질수록 **화면 전체를 대상으로 계산하는 비용**이 커질 수 있습니다.
- 현재 구현은 **기능 검증 및 학습용 예제**에 가깝고, 대규모 실사용 최적화 버전은 아닙니다.
